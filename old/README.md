
Photography Portfolio Website
Overview

This project is a responsive personal photography portfolio website designed to showcase collections of personal, nature, and urban photography. It emphasizes clean design, usability, and adaptability across multiple screen sizes.

Project Files

personal.html
Contains the main website structure, navigation menu, and photo gallery layout.

k.css
Manages styling, layout, and responsive behavior using CSS media queries.

Technologies Used

HTML5 – Semantic structure and content organization

CSS3 – Styling, layout, and responsive design

CSS Grid – Responsive photo gallery layout

Media Queries – Device-specific styling for mobile, tablet, and desktop

GitHub Pages – Static site hosting and deployment

Features

Responsive photo gallery using a grid-based layout

Intuitive navigation menu with hover effects

Mobile-first design approach

Clean, professional typography using the Arial font family

Responsive Design

The website adapts seamlessly to different screen sizes:

Mobile (below 480px)
Default layout optimized for small screens

Tablet (480px – 767px)
Brown background with white text for improved readability

Desktop (768px and above)
Wheat background with black text for a clean, classic appearance

Layout Structure

The photo gallery uses a three-column grid on larger screens and dynamically adjusts for smaller devices. Each section includes images with descriptive text highlighting the photographer’s work.

Customization

Add or replace photos by editing personal.html

Update text content and navigation links directly in the HTML

Modify colors, spacing, fonts, and breakpoints in k.css

Deployment & Setup (GitHub Pages)
Local Setup

Download or clone this repository.

Ensure personal.html and k.css are located in the same directory.

Open personal.html in any modern web browser to preview the site locally.

No external dependencies or build tools are required.


Store all image files inside the images folder and update image paths in the HTML as needed.

Deploying to GitHub Pages

Create a new repository on GitHub.

Upload or push all project files to the repository.

Navigate to Settings → Pages.

Under Source, select:

Branch: main

Folder: /root

Save the changes.
